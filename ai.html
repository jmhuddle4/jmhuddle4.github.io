Artificially Intelligent Tic Tac Toe C++


Spring 2015

Tic-tac-toe can be played on a general nrRows x nrCols board where a player wins by having nrToWin of its elements consecutively in a row, column, diagonal, or anti-diagonal. The traditional tic-tac-toe corresponds to the case where nrRows = nrCols = nrToWin = 3. This project, implements an evaluation function for the general tic-tac-toe game.

The interface is: nrRows nrCols nrToWin player1Type player2Type

Where player1Type is a string corresponding to either ‘human’ or ‘AI’ and player2Type is a string corresponding to either ‘human’ or ‘AI’.

The program displays the board each move, if human it prompts the user for input, if AI the program will automatically make its move based on the Evaluation Function.

On a 3x3 board, the program never loses. In fact, the program will never lose when nrRows = nrCols = nrToWin.

Artificially Intelligent Maze Solver Java


Spring 2015

1) Maze Generation
The program takes the following arguments from the command line:

nrRows nrCols maze.txt

Where nrRows is the number of rows, nrCols is the number of columns, and maze.txt is the name of the output file. The program generates random maze details which are written to the file maze.txt. The format of the output file is as follows:

- First line contains the number of rows and the number of columns
- Second line contains the number of remaining lines in the file
- Each line from the third to the last contains the row index, the column index, and a string describing which walls are up (the line 2 3 LRU indicates that the cell in row 2 and column 3 has the left, right, and up walls).
- L for the left wall
- R for the right wall
- U for the up wall
- D for the down wall


2) Maze Search
The program takes the following arguments from the command line:
method maze.txt rstart cstart rend cend path.txt

Where method is the name of the search method, maze.txt is the name of the file with the maze details, rstart is the row index of the start position, cstart is the column index of the start position, rend is the row index of the end position, cend is the column index of the end position.

The method name is as follows: DFS for depth-first search or BFS for breadth-first search. The program reads the file maze.txt and searches for a solution from (rstart, cstart) to (rend, cend). It then writes the solution to the file path.txt. The format of the output file is as follows:

- first line contains the number of elements in the path
- each line from the second to the last contains the row index and the column index of the i-th cell in the path

The text files are then translated by a function which outputs the maze and path to a GUI.
